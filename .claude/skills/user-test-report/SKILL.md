# user-test-report — 用户测试报告

## 技能目的

创建或分析用户测试反馈，将零散的用户测试数据转化为结构化的改进建议报告。通过收集测试数据、分析反馈模式、识别共性问题并按严重度排序，为产品迭代提供数据驱动的改进方向。

## 参数说明

- `[测试轮次]`：用户测试轮次标识，例如 `round-1`、`beta-2`，用于定位测试数据与生成报告命名。

## 分阶段工作流

### 阶段 1：收集用户测试数据

- **输入**：`[测试轮次]` 参数
- **处理**：
  1. 使用 Glob 查找与测试轮次相关的数据文件（`docs/user-tests/[轮次]/*.md`、`*.csv`、`*.json`）
  2. 使用 Read 读取用户测试记录、问卷结果与访谈笔记
  3. 使用 Grep 搜索已有的用户反馈汇总文档
  4. 通过 AskUserQuestion 补充缺失的测试数据或确认数据来源
- **输出**：用户测试原始数据集

### 阶段 2：分析反馈模式

- **输入**：原始数据集
- **处理**：
  1. 使用 Grep 按关键词分类反馈（功能缺陷、易用性问题、性能反馈、功能缺失、体验良好）
  2. 统计每类反馈的出现频次
  3. 使用 Read 深入分析高频反馈的具体内容
  4. 识别反馈之间的关联模式与潜在根因
- **输出**：反馈模式分析结果

### 阶段 3：识别共性问题

- **输入**：反馈模式分析
- **处理**：
  1. 将出现频次超过阈值的反馈标记为共性问题
  2. 将影响核心流程的反馈标记为关键问题
  3. 将多位测试者独立提出的相同反馈标记为高置信度问题
  4. 汇总共性问题清单
- **输出**：共性问题清单

### 阶段 4：按严重度排序

- **输入**：共性问题清单
- **处理**：
  1. 定义严重度维度：影响范围、发生频率、业务影响、用户情绪强度
  2. 为每个问题在各维度评分
  3. 计算综合严重度得分并排序
  4. 通过 AskUserQuestion 与用户确认排序与优先级
- **输出**：按严重度排序的问题列表

### 阶段 5：生成改进建议报告

- **输入**：排序后的问题列表
- **处理**：
  1. 为每个高优先级问题提出具体改进建议
  2. 关联建议到相关需求文档或用户故事
  3. 标注建议的预期实施团队（前端 / 后端 / 设计）
  4. 使用 Write 写入报告至 `docs/reports/user-test-[轮次].md`
- **输出**：用户测试改进建议报告
