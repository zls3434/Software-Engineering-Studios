# content-audit — 内容审计与差距分析

## 技能目的

对比软件需求规格说明（SRS）中指定的内容与项目已实现的内容，发现需求与实现之间的差距。通过映射需求项到实现代码，标记缺失功能与多余功能，生成结构化差距报告，为后续补全或裁剪提供依据。

## 参数说明

本技能无参数。自动扫描全部 SRS 文档与源码进行对比。

## 分阶段工作流

### 阶段 1：读取 SRS 需求列表

- **输入**：项目目录
- **处理**：
  1. 使用 Glob 查找 `docs/srs/*.md` 获取全部需求规格文档
  2. 使用 Read 逐个读取 SRS 文档，提取需求项 ID、标题与验收标准
  3. 使用 Grep 提取每个 SRS 中的功能需求列表与用户故事
  4. 汇总形成完整的需求项清单
- **输出**：SRS 需求项清单

### 阶段 2：扫描已实现代码

- **输入**：项目源码目录
- **处理**：
  1. 使用 Glob 扫描源码目录结构（前端组件、后端服务、API 路由）
  2. 使用 Read 抽样读取关键实现文件确认功能存在性
  3. 使用 Grep 搜索需求项 ID 或功能关键词在代码中的引用
  4. 汇总形成已实现功能清单
- **输出**：已实现功能清单

### 阶段 3：映射需求到实现

- **输入**：需求项清单 + 已实现功能清单
- **处理**：
  1. 逐项将 SRS 需求与已实现功能进行匹配
  2. 为每个需求项标注实现状态：已实现、部分实现、未实现、未追溯
  3. 使用 Grep 交叉验证部分实现项的覆盖程度
  4. 记录映射关系与匹配依据
- **输出**：需求-实现映射矩阵

### 阶段 4：标记缺失或多余功能

- **输入**：映射矩阵
- **处理**：
  1. 标记在 SRS 中存在但代码中未实现的需求项为缺失功能
  2. 标记在代码中存在但 SRS 中无对应需求的功能为多余功能
  3. 标记部分实现项的具体缺失部分
  4. 按严重度对缺失项排序（阻塞性 > 重要 > 次要）
- **输出**：缺失与多余功能清单

### 阶段 5：生成差距报告

- **输入**：全部阶段产物
- **处理**：
  1. 汇总映射矩阵与缺失/多余清单
  2. 统计覆盖率指标（已实现 / 部分实现 / 未实现百分比）
  3. 提供补全建议与裁剪建议
  4. 使用 Write 写入报告至 `docs/reports/content-audit.md`
- **输出**：内容差距报告

## 协作协议引用

- 遵循 `docs/examples/COLLABORATIVE-DESIGN-PRINCIPLE.md` 用户驱动协作协议
- 参考 `docs/templates/requirement-spec.md` 需求规格模板
- 参考 `docs/templates/architecture-traceability.md` 追溯矩阵模板
- 审计为只读分析，不修改源码与需求文档

## 推荐下一步

根据差距报告补全缺失功能，使用 `/team-frontend` 或 `/team-backend` 编排实现。多余功能需确认是否裁剪，使用 `/review-all-srs` 更新需求文档。补全后重新运行本技能验证覆盖率提升。
