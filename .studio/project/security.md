# 安全注意事项

本文件是平台无关的安全规范源，定义所有 Agent 和开发流程必须遵守的安全规则。

## 1. 密钥管理

### 1.1 禁止硬编码密钥

- **禁止**在源代码中硬编码任何密钥、令牌、密码或敏感凭证
- **禁止**在配置文件中明文存储密钥
- **禁止**在测试代码中使用真实的生产密钥
- **禁止**在日志、错误信息或提交信息中暴露密钥

### 1.2 API 密钥通过环境变量注入

- 所有密钥必须通过环境变量注入
- 环境变量通过 `.env` 文件管理（本地开发）或密钥管理服务（生产环境）
- 应用程序通过 `process.env`、`os.environ` 等方式读取环境变量
- 敏感配置项在运行时动态加载，不持久化到代码库

### 1.3 .env 不纳入版本控制

- `.env` 文件必须添加到 `.gitignore`
- 提交 `.env.example` 作为模板，仅包含变量名和占位符
- **禁止**提交包含真实密钥的 `.env` 文件
- `validate-assets.sh` Hook 会检查是否有 `.env` 文件被意外提交

## 2. 禁止操作

### 2.1 禁止 `rm -rf`

- **禁止**在脚本或命令中使用 `rm -rf` 进行递归强制删除
- 删除操作必须明确指定目标路径，不得使用通配符递归
- 需要清理目录时，使用明确的路径和受控的删除命令
- 临时文件清理应使用受控的清理脚本

### 2.2 禁止 `git push --force`

- **禁止**对受保护分支（`main`、`release/*`）执行 `git push --force`
- `validate-push.sh` Hook 会拦截对受保护分支的强制推送
- 需要修改历史时，使用 `git rebase` 后正常推送
- 热修复通过 `hotfix/` 分支走正常合并流程

### 2.3 禁止读取 `.secrets` 目录

- **禁止**任何 Agent 或脚本读取 `.secrets/` 目录内容
- `.secrets/` 目录用于存放本地敏感文件，不纳入版本控制
- 需要密钥时通过环境变量获取，不直接读取密钥文件

## 3. 数据库安全

### 3.1 禁止破坏性操作

- **禁止**在生产环境直接执行 `DROP`、`DELETE`、`TRUNCATE` 操作
- 破坏性操作必须通过迁移脚本执行，并经过 `database-engineer` 审批
- 数据删除使用软删除（标记删除标志）优先于硬删除

### 3.2 禁止 SQL 注入

- **禁止**在业务逻辑中直接拼接 SQL 字符串
- 必须使用参数化查询或 ORM 框架
- 用户输入必须经过验证和转义后才能用于查询

### 3.3 禁止存储明文敏感数据

- 密码必须使用 bcrypt/argon2 等算法哈希存储
- 令牌（Token）必须加密存储
- 不得在数据库中存储明文的 API 密钥或第三方凭证

## 4. 前端安全

- **禁止**在客户端代码中存储任何服务端密钥
- **禁止**将敏感配置内联到前端打包产物中
- 用户输入必须经过验证和转义，防止 XSS 攻击
- 跨域请求必须配置正确的 CORS 策略

## 5. 依赖安全

- **禁止**使用已弃用或已知存在安全漏洞的依赖
- 定期通过 `npm audit`、`pip audit`、`govulncheck` 等工具检查依赖安全
- 依赖升级通过 `chore(deps)` 提交，经测试验证后合并
- 安全扫描是 CI 流水线的必需阶段

## 6. 访问控制

- `.claude/` 或 `.studio/` 下的配置文件只能由 `chief-architect` 审批修改
- 任何 Agent 不得单方面修改不属于自己领域的配置文件
- 生产环境部署需通过发布门禁（`/gate-check`）
- 安全审计由 `security-lead` 和 `security-engineer` 负责

## 7. 安全事件响应

- 发现安全漏洞时立即通知 `security-lead`
- 安全事件按 `docs/templates/incident-response.md` 模板记录
- 严重漏洞需创建热修复分支（`hotfix/`）紧急处理
- 事后进行复盘，记录在 `docs/templates/post-mortem.md` 中

## 8. 安全检查清单

- [ ] 所有密钥通过环境变量注入，无硬编码
- [ ] `.env` 文件已添加到 `.gitignore`
- [ ] 无 `.env` 文件被提交到版本控制
- [ ] 无 `rm -rf` 命令在脚本中使用
- [ ] 无 `git push --force` 对受保护分支执行
- [ ] 数据库操作使用参数化查询或 ORM
- [ ] 密码使用安全哈希算法存储
- [ ] 依赖无已知安全漏洞
- [ ] CI 安全扫描阶段已配置
- [ ] CORS 策略已正确配置
