# 设计 Agent 协作协议

> 本协议定义设计 Agent 的标准协作周期：**提问 → 选项 → 草稿 → 批准**。
> 每个 Agent 在执行设计任务时必须遵循此周期，以确保产出可追溯、可决策、可审批。

## 1. 协作周期概览

```
提问（Clarify）→ 选项（Options）→ 草稿（Draft）→ 批准（Approve）
```

- **提问**：在开始设计前，向相关方提出澄清问题，消除歧义。
- **选项**：基于澄清结果，提出 2-4 个候选方案并比较权衡。
- **草稿**：选定方向后，产出可评审的草稿文档或原型。
- **批准**：提交草稿至决策方，获得明确批准或修改意见后进入下一阶段。

## 2. 澄清问题模板

在进入选项阶段前，Agent 应使用以下模板提出澄清问题：

```markdown
### 澄清问题清单

**背景**：[此处填写任务/需求的简要背景]

**问题**：
1. [此处填写问题 1，聚焦于目标与范围]
2. [此处填写问题 2，聚焦于用户与场景]
3. [此处填写问题 3，聚焦于约束与优先级]
4. [此处填写问题 4，聚焦于成功标准]

**期望回复时限**：[此处填写时限，如 24 小时]

**未回复处理**：[此处填写若未及时回复的默认处理方式，如按假设推进并标记假设]
```

```markdown
- **草稿版本**：[此处填写，如 v0.1]
- **草稿位置**：[此处填写文档/原型链接]
- **摘要**：[此处填写 3-5 句摘要]
- **待决策点**：[此处填写需要决策方拍板的关键点]
```

### 4.2 审批决策

决策方需在以下选项中明确选择一项：

- [ ] **批准**：草稿可进入下一阶段，无需修改。
- [ ] **有条件批准**：草稿基本可行，需按所列修改意见调整后继续。
- [ ] **驳回**：草稿方向有误，需重新进入"选项"阶段。
- [ ] **暂缓**：因外部依赖未就绪，暂时挂起。

**修改意见**：[此处填写具体修改意见]

**决策人**：[此处填写]
**决策日期**：[此处填写 YYYY-MM-DD]
