# /regression-suite — 将测试覆盖率映射到 SRS 关键路径，标记覆盖差距，维护并更新回归测试套件。

## 触发方式
在 Cascade 对话框中输入 /regression-suite 触发此工作流。

## 工作流内容
# regression-suite — 回归测试套件维护

## 技能目的

将现有测试覆盖率映射到 SRS 关键路径，标记未覆盖的差距，生成补充回归测试用例并更新回归套件，确保每次变更不会引入回归缺陷。

## 参数说明

本技能无参数。自动读取 SRS 与现有测试。

## 分阶段工作流

### 阶段 1：读取 SRS 关键路径

- **输入**：项目目录
- **处理**：
  1. 使用 Glob 查找 `docs/srs/**/*.md` 获取所有需求规格说明
  2. 使用 Read 读取每个 SRS 的关键路径与用例章节
  3. 使用 Grep 搜索标记为 `@critical`、`@regression`、`@p0` 的需求项
  4. 提取路径编号、入口条件、主流程、异常流程
- **输出**：SRS 关键路径清单

### 阶段 2：分析现有测试覆盖

- **输入**：SRS 关键路径清单
- **处理**：
  1. 使用 Glob 查找测试文件（`tests/`、`e2e/`、`__tests__/`）
  2. 使用 Bash 运行覆盖率收集命令（带 `--coverage`）
  3. 解析覆盖率报告，得到文件/函数/分支覆盖矩阵
  4. 使用 Grep 将测试用例与 SRS 路径编号关联（注释或标签）
- **输出**：覆盖率矩阵

### 阶段 3：标记未覆盖路径

- **输入**：覆盖率矩阵
- **处理**：
  1. 逐条比对 SRS 关键路径与已覆盖项
  2. 标注三类差距：完全未覆盖、部分覆盖（缺异常流程）、覆盖但未断言
  3. 评估每条差距的回归风险等级
  4. 生成差距清单并排序
- **输出**：覆盖差距清单

### 阶段 4：生成回归测试用例

- **输入**：覆盖差距清单
- **处理**：
  1. 为每条高优先级差距生成测试用例骨架
  2. 用例包含：前置条件、步骤、期望结果、数据准备
  3. 标注用例与 SRS 路径编号的映射关系
  4. 使用 Write 写入 `tests/regression/` 目录
- **输出**：新增回归用例文件

### 阶段 5：更新套件

- **输入**：新增回归用例文件
- **处理**：
  1. 更新测试运行配置，将回归用例纳入套件
  2. 使用 Bash 运行更新后的套件验证用例可执行
  3. 更新 `docs/test/regression-coverage.md` 覆盖映射文档
  4. 使用 Write 生成套件变更说明
- **输出**：更新后的回归套件

## 协作协议引用

- 遵循 `.claude/docs/coding-standards.md` 编码规范
- 回归用例写入 `tests/regression/` 目录
- 覆盖映射文档写入 `docs/test/` 目录
- 套件变更需通过冒烟验证

## 推荐下一步

使用 `/smoke-check` 验证更新后的套件。使用 `/test-evidence-review` 审查测试质量。使用 `/gate-check` 在阶段切换时确认回归覆盖达标。
